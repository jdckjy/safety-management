# 제주 헬스케어타운 단지 조성 및 자산관리 시스템: 기술 문서

> 최종 업데이트: 2024년 5월 21일

## 1. 프로젝트 개요

본 문서는 제주 헬스케어타운의 단지 조성 현황과 완공 후 자산(세대)의 임대 및 분양 관리를 효율화하기 위해 구축된 '업무관리 시스템'의 기술적 사양과 아키텍처, 핵심 기능 구현 방식을 상세히 기술합니다.

## 2. 핵심 목표

- **데이터 중심 의사결정**: 분산된 엑셀 파일과 문서를 중앙화된 데이터베이스로 통합하여, 모든 의사결정이 실시간 데이터에 기반하도록 지원합니다.
- **업무 자동화 및 효율화**: 반복적인 보고서 작성, 데이터 취합, 현황 분석 등의 업무를 자동화하여 휴먼 에러를 줄이고 업무 효율을 극대화합니다.
- **직관적 데이터 시각화**: 복잡한 단지 조성 현황과 임대/수익 데이터를 사용자가 쉽게 이해할 수 있는 인터랙티브 대시보드와 도면으로 시각화합니다.

## 3. 핵심 아키텍처 및 기술 스택

- **언어**: TypeScript
- **UI 프레임워크**: React
- **UI 컴포넌트**: shadcn/ui
- **데이터 시각화**: Chart.js
- **상태 관리**: React Context API
- **백엔드/데이터베이스**: Google Firestore
- **빌드 도구**: Vite
- **호스팅**: Firebase Hosting
- **버전 관리**: Git / GitHub

## 11. 시스템 안정화 및 배포 (2024.05.21 추가)

### 11.1. 개요

초기 개발 단계에서 발생했던 여러 기술적 부채와 오류들을 해결하고, 시스템 전반의 안정성을 확보하기 위한 리팩토링 및 버그 수정 작업을 진행했습니다. 모든 기능의 정상 작동을 확인한 후, 최종 프로덕션 빌드 및 Firebase Hosting을 통한 배포를 완료했습니다.

### 11.2. 주요 개선 사항

#### 1. Firebase 설정 통합 및 리팩토링

- **문제점**: 기존에는 Firebase 인증(`auth`)과 데이터베이스(`db`) 초기화 코드가 각각 `src/firebaseConfig.ts`와 `src/firebase.ts`에 분산되어 있었습니다. 이로 인해 Firebase 앱이 중복으로 초기화되고, 모듈 간 순환 참조 및 임포트 오류가 발생하는 등 잠재적인 문제가 있었습니다.
- **해결 방안**: 모든 Firebase 관련 초기화 로직을 `src/firebase.ts` 파일 하나로 통합했습니다. `firebaseConfig.ts` 파일을 삭제하고, `LoginPage.tsx` 등 `auth` 객체를 사용하던 모든 컴포넌트의 임포트 경로를 `src/firebase`로 통일하여 문제를 해결했습니다.
- **기대 효과**: 코드의 중복을 제거하고 프로젝트 구조를 단순화했으며, `auth` 관련 모듈을 찾을 수 없다는(does not provide an export named 'auth') 오류를 근본적으로 해결했습니다.

#### 2. TypeScript 타입 시스템 안정화

- **문제점**: 수입/지출 내역을 관리하는 `LeaseAnalysisPage.tsx`에서 항목을 수정할 때, `IncomeItem`과 `ExpenseItem`의 `category` 속성 타입(각각 `IncomeCategory`, `ExpenseCategory`)이 서로 달라 지속적으로 타입 충돌 오류가 발생했습니다.
- **해결 방안**: `src/types.ts` 파일에서 `IncomeItem`과 `ExpenseItem` 인터페이스의 `category` 속성 타입을 보다 범용적인 `string`으로 변경했습니다. 이를 통해 두 타입을 포괄하는 함수(`handleUpdateFinancialItem`)에서 더 이상 타입 불일치 문제가 발생하지 않도록 조치했습니다.
- **기대 효과**: 반복적으로 발생하던 TypeScript 컴파일 오류를 해결하여 코드의 안정성과 신뢰도를 높였습니다.

#### 3. 기타 버그 수정 및 최종 배포

- 개발 과정에서 발견된 다수의 사소한 버그(JSX 태그 오타, 문자열 처리 오류 등)를 수정하여 코드 품질을 향상시켰습니다.
- `npm run build` 명령어를 통해 모든 코드를 최적화한 프로덕션 빌드를 생성했습니다.
- 최종적으로 생성된 `dist` 폴더를 Firebase Hosting에 배포하여 실제 사용자가 접근할 수 있는 라이브 환경에 반영을 완료했습니다.
