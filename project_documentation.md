## 12. AI 브리핑 기능 오류 해결 및 API 키 유출 사건 (2024.05.21 추가)

### 12.1. 문제 상황

신규 개발된 "AI 일일 브리핑" 기능이 프로덕션 환경에서 지속적으로 실패하며, 다음과 같은 오류들을 순차적으로 발생시켰습니다.

- **초기 오류:** `[404 Not Found] models/gemini-pro is not found for API version v1beta`
- **중간 오류:** `[404 Not Found] models/gemini-1.5-pro-latest is not found for API version v1beta`
- **최종 오류:** `[403 Forbidden] Your API key was reported as leaked. Please use another API key.`

### 12.2. 잘못된 진단과 해결 시도

초기 `404 Not Found` 오류를 바탕으로, 문제의 원인이 '잘못된 모델 이름' 또는 '오래된 API 호출 방식'에 있다고 잘못 판단했습니다. 이로 인해 다음과 같은 불필요하고 비효율적인 해결 시도를 반복했습니다.

1.  **API 호출 방식 변경:** `fetch`를 사용한 직접적인 REST API 호출 시도.
2.  **모델 이름 변경:** `gemini-pro` -> `gemini-1.5-pro-latest` -> `gemini-3-flash-preview` 등 여러 모델 이름으로 변경하며 테스트.
3.  **라이브러리 재적용:** 다시 `@google/generative-ai` 라이브러리를 사용하는 방식으로 복귀.

이 모든 시도는 문제의 근본 원인을 해결하지 못했으며, 배포와 테스트를 반복하며 시간만 지연시키는 결과를 낳았습니다.

### 12.3. 근본 원인: API 키 유출

마침내 수신된 `403 Forbidden: API key was reported as leaked` 메시지를 통해 모든 문제의 근본 원인이 **API 키의 유출 및 차단**에 있었음을 명확히 확인했습니다.

- **사건의 재구성:** 개발 초기 단계에서, API 키가 포함된 `.env` 파일이 실수로 공개 GitHub 저장소에 커밋(commit) 및 푸시(push) 되었습니다.
- **Google의 자동 차단:** Google의 보안 시스템은 공개 저장소에서 API 키를 감지하고, 무단 사용을 방지하기 위해 해당 키를 자동으로 비활성화(차단)했습니다.
- **결론:** 차단된 API 키를 사용했기 때문에, 어떤 모델 이름이나 API 호출 방식을 사용하더라도 Google 서버로부터 접근이 거부(`403 Forbidden`)되거나, 해당 키로 접근 가능한 모델이 없다는 `404 Not Found` 오류가 발생했던 것입니다.

### 12.4. 최종 해결 방안

문제의 근본 원인을 파악한 후, 다음과 같이 명확하고 올바른 조치를 수행했습니다.

1.  **신규 API 키 발급:** Google Cloud Console에서 유출된 기존 키를 폐기하고, 새로운 API 키를 발급받았습니다.
2.  **.env 파일 업데이트:** 프로젝트의 `.env` 파일에 새로 발급받은 API 키를 적용했습니다. 코드는 `import.meta.env.VITE_API_KEY`를 통해 이 값을 참조하므로, 코드 수정은 불필요했습니다.
3.  **.gitignore 파일 수정:** **가장 중요한 조치**로, `.gitignore` 파일에 `.env`와 `.env.local`을 추가하여 다시는 민감한 환경 변수 파일이 Git 버전 관리 대상에 포함되거나 원격 저장소에 올라가지 않도록 방지했습니다.

### 12.5. 교훈

이번 사건은 다음과 같은 중요한 보안 원칙을 다시 한번 상기시키는 계기가 되었습니다.

- **비밀은 절대 코드에 포함하지 않는다 (Never commit secrets):** API 키, 비밀번호 등 모든 민감 정보는 환경 변수(`.env`) 파일을 통해 관리해야 합니다.
- **.gitignore는 프로젝트의 보안 생명선이다:** 프로젝트 시작 시, `.env`, `node_modules` 등 민감하거나 불필요한 파일/폴더를 `.gitignore`에 가장 먼저 등록하는 습관이 매우 중요합니다.
