# 제주 헬스케어타운 단지 조성 및 자산관리 시스템: 기술 문서

> 최종 업데이트: 2026년 3월 24일

## 1. 프로젝트 개요

본 문서는 제주 헬스케어타운의 단지 조성 현황과 완공 후 자산(세대)의 임대 및 분양 관리를 효율화하기 위해 구축된 '업무관리 시스템'의 기술적 사양과 아키텍처, 핵심 기능 구현 방식을 상세히 기술합니다.

## 2. 핵심 목표

- **데이터 중심 의사결정**: 분산된 엑셀 파일과 문서를 중앙화된 데이터베이스로 통합하여, 모든 의사결정이 실시간 데이터에 기반하도록 지원합니다.
- **업무 자동화 및 효율화**: 반복적인 보고서 작성, 데이터 취합, 현황 분석 등의 업무를 자동화하여 휴먼 에러를 줄이고 업무 효율을 극대화합니다.
- **직관적 데이터 시각화**: 복잡한 단지 조성 현황과 임대/수익 데이터를 사용자가 쉽게 이해할 수 있는 인터랙티브 대시보드와 도면으로 시각화합니다.

## 3. 핵심 아키텍처 및 기술 스택

- **언어**: TypeScript
- **UI 프레임워크**: React
- **UI 컴포넌트**: shadcn/ui
- **데이터 시각화**: Chart.js
- **상태 관리**: React Context API
- **백엔드/데이터베이스**: Google Firestore
- **빌드 도구**: Vite
- **호스팅**: Firebase Hosting
- **버전 관리**: Git / GitHub

## 4. 데이터 모델링 (Firestore)

- **`tenantUnits`**: 각 세대(호실)의 상세 정보 (면적, 임대료, 상태 등)
- **`complexFacilities`**: 단지 내 주요 시설물 정보
- **`teamMembers`**: 프로젝트 관련 인원 정보
- **`activities`**: 프로젝트 활동 로그
- **`kpis`**: 핵심 성과 지표

## 5. 핵심 기능 구현

### 5.1. 실시간 대시보드
- **`LeaseStatusWidget`**: 전체 임대율, 공실률, 예상 월수익 등 핵심 지표를 실시간으로 시각화합니다.
- **`TaskStatusPieChart`**: 프로젝트의 업무 진행 상태를 파이 차트로 보여줍니다.

### 5.2. 인터랙티브 도면
- **`FloorPlanBrowser`**: 층별 도면을 SVG로 렌더링하고, 각 호실을 클릭하여 상세 정보를 확인하고 상태(임대, 공실 등)를 변경할 수 있습니다.

### 5.3. 활동 및 성과 관리
- **`ActivityManager`**: 프로젝트와 관련된 모든 활동(회의, 결정사항, 이슈)을 기록하고 추적합니다.
- **`KPIManager`**: 핵심 성과 지표를 설정하고, 목표 대비 실적을 관리합니다.

## 6. 주요 UI 컴포넌트

- **`shadcn/ui`** 를 기반으로 일관된 디자인 시스템을 구축했습니다.
- **`Card`**: 정보 블록
- **`Table`**: 데이터 표
- **`Modal`**: 상세 정보 및 수정/입력 폼
- **`Button`, `Input`, `Select`**: 표준 인터랙션 요소

## 7. 상태 관리

- **React Context API** 를 사용하여 전역 상태(프로젝트 데이터, 사용자 정보 등)를 관리합니다.
- **`ProjectDataProvider`**: Firestore로부터 프로젝트 데이터를 비동기적으로 로드하고, 앱 전반에 제공하는 역할을 합니다.

## 8. 빌드 및 배포

- **Vite** 를 사용하여 빠른 개발 서버와 최적화된 프로덕션 빌드를 생성합니다.
- **`npm run build`** 명령으로 `dist` 디렉토리에 정적 파일을 생성합니다.
- **Firebase Hosting** 을 통해 `dist` 디렉토리를 정적 웹사이트로 배포합니다.

## 9. Firebase 연동

- **`src/firebase.ts`**: Firebase 앱 초기화 및 Firestore, Auth 인스턴스를 생성하는 통합 설정 파일입니다.
- 모든 컴포넌트는 이 파일을 통해 Firebase 서비스에 접근합니다.

## 10. 신규 기능: 공실 해소 시뮬레이터 (2026.03.24 추가)

### 10.1. 기능 개요

`src/components/LeaseSimulator.tsx` 컴포넌트를 통해 구현된 기능으로, 사용자가 직접 조건을 설정하여 공실 해소에 따른 임대 수익 변화를 예측해볼 수 있는 도구입니다.

### 10.2. 핵심 로직

- **입력**: 사용자는 슬라이더를 이용해 '추가 임대 면적'을 직관적으로 조절할 수 있습니다.
- **처리**: `useLeaseAnalytics` 훅의 `runLeaseSimulation` 함수를 호출하여, 추가 임대 면적을 반영했을 경우의 새로운 임대율과 예상 월 수익을 계산합니다.
- **출력**: 시뮬레이션 결과를 '예상 임대율'과 '월 예상 수익' 카드로 명확하게 표시하며, 기존 대비 변화량을 퍼센트(%)와 금액(억)으로 함께 보여주어 효과적인 의사결정을 지원합니다.

## 11. 시스템 안정화 및 배포 (2026.03.24 추가)

### 11.1. 개요

초기 개발 단계에서 발생했던 여러 기술적 부채와 오류들을 해결하고, 시스템 전반의 안정성을 확보하기 위한 리팩토링 및 버그 수정 작업을 진행했습니다. 모든 기능의 정상 작동을 확인한 후, 최종 프로덕션 빌드 및 Firebase Hosting을 통한 배포를 완료했습니다.

### 11.2. 주요 개선 사항

#### 1. Firebase 설정 통합 및 리팩토링

- **문제점**: 기존에는 Firebase 인증(`auth`)과 데이터베이스(`db`) 초기화 코드가 각각 `src/firebaseConfig.ts`와 `src/firebase.ts`에 분산되어 있었습니다.
- **해결 방안**: 모든 Firebase 관련 초기화 로직을 `src/firebase.ts` 파일 하나로 통합하여 `auth` 관련 모듈 임포트 오류를 근본적으로 해결했습니다.

#### 2. TypeScript 타입 시스템 안정화

- **문제점**: 수입/지출 내역 관리에서 `IncomeItem`과 `ExpenseItem`의 타입 불일치로 인한 오류가 발생했습니다.
- **해결 방안**: `category` 속성 타입을 `string`으로 통일하여 타입 충돌을 해결했습니다.

## 12. AI 브리핑 기능 오류 해결 및 API 키 유출 사건 (2026.03.24 추가)

### 12.1. 문제 상황

"AI 일일 브리핑" 기능이 `404 Not Found` 및 `403 Forbidden` 오류를 발생시키며 실패했습니다.

### 12.2. 근본 원인: API 키 유출

모든 문제의 근본 원인은 **API 키의 유출 및 차단**에 있었습니다. 개발 초기, API 키가 포함된 `.env` 파일이 공개 GitHub 저장소에 커밋되어 Google에 의해 자동으로 키가 차단되었습니다.

### 12.3. 최종 해결 방안

1.  **신규 API 키 발급**: Google Cloud Console에서 새 API 키를 발급받아 `.env` 파일에 적용했습니다.
2.  **.gitignore 수정**: `.gitignore` 파일에 `.env`를 추가하여 민감 정보가 다시는 유출되지 않도록 조치했습니다.

### 12.4. 교훈

- **비밀은 절대 코드에 포함하지 않는다 (Never commit secrets).**
- **.gitignore는 프로젝트의 보안 생명선이다.**
