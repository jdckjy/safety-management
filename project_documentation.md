# 제주 헬스케어타운 단지 조성 및 자산관리 시스템: 기술 문서

> 최종 업데이트: 2026년 3월 27일

## 1. 프로젝트 개요

본 문서는 제주 헬스케어타운의 단지 조성 현황과 완공 후 자산(세대)의 임대 및 분양 관리를 효율화하기 위해 구축된 '업무관리 시스템'의 기술적 사양과 아키텍처, 핵심 기능 구현 방식을 상세히 기술합니다.

## 2. 핵심 목표

- **데이터 중심 의사결정**: 분산된 엑셀 파일과 문서를 중앙화된 데이터베이스로 통합하여, 모든 의사결정이 실시간 데이터에 기반하도록 지원합니다.
- **업무 자동화 및 효율화**: 반복적인 보고서 작성, 데이터 취합, 현황 분석 등의 업무를 자동화하여 휴먼 에러를 줄이고 업무 효율을 극대화합니다.
- **직관적 데이터 시각화**: 복잡한 단지 조성 현황과 임대/수익 데이터를 사용자가 쉽게 이해할 수 있는 인터랙티브 대시보드와 도면으로 시각화합니다.

## 3. 핵심 아키텍처 및 기술 스택

- **언어**: TypeScript
- **UI 프레임워크**: React
- **UI 컴포넌트**: shadcn/ui
- **데이터 시각화**: Recharts (Chart.js에서 변경)
- **상태 관리**: React Context API
- **백엔드/데이터베이스**: Google Firestore
- **빌드 도구**: Vite
- **호스팅**: Firebase Hosting
- **버전 관리**: Git / GitHub

## 4. 데이터 모델링 (Firestore)

- **`tenantUnits`**: 각 세대(호실)의 상세 정보 (면적, 임대료, 상태 등)
- **`complexFacilities`**: 단지 내 주요 시설물 정보
- **`teamMembers`**: 프로젝트 관련 인원 정보
- **`activities`**: 프로젝트 활동 로그
- **`kpis`**: 핵심 성과 지표
- **`monthly_reports`**: 월간 운영 데이터 (JSON 기반에서 Firestore로 통합)

## 18. 수익 분석 기능 개편 및 대시보드 정리 (2026.03.27 추가)

### 18.1. 기능 목표

사용자 피드백을 반영하여 메인 대시보드의 정보 과밀도를 낮추고, 재무 관련 분석 기능을 더욱 적합한 위치로 재배치하는 것을 목표로 합니다. 이를 통해 사용자는 각 페이지의 명확한 목적에 집중할 수 있으며, 전체 시스템의 사용성이 향상됩니다.

### 18.2. 핵심 변경 사항

#### 1. 대시보드에서 재무 분석 기능 분리

- **문제점**: 메인 대시보드에 연간 업무진척도, 임대 현황, 최근 활동 피드, 재무 요약 차트 등 너무 많은 정보가 혼재하여 사용자가 핵심 지표를 파악하기 어려웠습니다.
- **해결 방안**:
    - `src/components/Dashboard.tsx` 파일에서 `FinancialSummary` 컴포넌트와 관련 임시 데이터(sampleIncome, sampleExpense)를 완전히 제거했습니다.
    - 이를 통해 메인 대시보드는 프로젝트의 전반적인 진행 상황(업무, 임대)에 집중할 수 있도록 단순화 및 최적화되었습니다.

#### 2. '수익 분석' 탭으로 재무 분석 기능 이전 및 고도화

- **문제점**: '임대 및 세대' 탭의 '수익 분석' 페이지에는 '시설별 임대율 현황'이라는 막대 차트가 있었으나, 이는 전체 임대율 현황과 중복되는 정보를 제공하며 활용도가 낮았습니다.
- **해결 방안**:
    - `src/pages/LeaseAnalysisPage.tsx` 파일에서 기존 `react-chartjs-2` 기반의 '시설별 임대율 현황' 막대 차트와 관련 코드를 모두 삭제했습니다.
    - 삭제된 차트의 위치에, 대시보드에서 가져온 `FinancialSummary` 컴포넌트를 삽입했습니다.
    - **가장 큰 개선점**은, 기존의 임시 데이터를 사용하는 대신, `LeaseAnalysisPage.tsx`가 이미 Firebase로부터 실시간으로 수신하고 있던 `incomeItems`와 `expenseItems` 상태를 `FinancialSummary` 컴포넌트의 props로 직접 전달한 것입니다.

### 18.3. 종합 결과

- **직관성 향상**: 메인 대시보드가 프로젝트 관리 중심으로 명확해졌으며, 재무 분석은 '수익 분석' 탭에서 심도 있게 수행할 수 있도록 역할이 명확히 분리되었습니다.
- **데이터 일관성 확보**: 기존의 임시 데이터 대신 실제 데이터베이스(Firestore)와 연동된 차트를 제공함으로써, 사용자는 항상 최신의 정확한 '월별 수입 및 지출' 현황을 확인할 수 있게 되었습니다.
- **시스템 효율성 증대**: 불필요한 차트 라이브러리(`react-chartjs-2`) 의존성을 제거하고, 페이지의 목적에 맞는 컴포넌트를 재사용함으로써 코드의 유지보수성과 효율성을 높였습니다.
