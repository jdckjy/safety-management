# 제주 헬스케어타운 단지 조성 및 자산관리 시스템: 기술 문서

> 최종 업데이트: 2024년 5월 20일

## 1. 프로젝트 개요

본 문서는 제주 헬스케어타운의 단지 조성 현황과 완공 후 자산(세대)의 임대 및 분양 관리를 효율화하기 위해 구축된 '업무관리 시스템'의 기술적 사양과 아키텍처, 핵심 기능 구현 방식을 상세히 기술합니다.

## 2. 핵심 목표

- **데이터 중심 의사결정**: 분산된 엑셀 파일과 문서를 중앙화된 데이터베이스로 통합하여, 모든 의사결정이 실시간 데이터에 기반하도록 지원합니다.
- **업무 자동화 및 효율화**: 반복적인 보고서 작성, 데이터 취합, 현황 분석 등의 업무를 자동화하여 휴먼 에러를 줄이고 업무 효율을 극대화합니다.
- **직관적 데이터 시각화**: 복잡한 단지 조성 현황과 임대/수익 데이터를 사용자가 쉽게 이해할 수 있는 인터랙티브 대시보드와 도면으로 시각화합니다.

## 3. 핵심 아키텍처 및 기술 스택

- **언어**: TypeScript (JavaScript의 상위 집합으로, 정적 타입을 통해 코드의 안정성과 유지보수성을 높입니다.)
- **UI 프레임워크**: React (컴포넌트 기반 아키텍처를 통해 재사용 가능하고 독립적인 UI를 구축합니다.)
- **UI 컴포넌트**: shadcn/ui (Radix UI와 Tailwind CSS를 기반으로, 접근성이 높고 커스터마이징이 용이한 컴포넌트 라이브러리입니다.)
- **데이터 시각화**: Chart.js (다양한 종류의 인터랙티브 차트를 손쉽게 구현할 수 있는 라이브러리입니다.)
- **상태 관리**: React Context API (`ProjectDataProvider`)
- **백엔드/데이터베이스**: Google Firestore (실시간 동기화를 지원하는 NoSQL 클라우드 데이터베이스)
- **빌드 도구**: Vite (빠른 개발 서버와 효율적인 빌드 시스템을 제공합니다.)
- **버전 관리**: Git / GitHub

### 3.2. 해결 방안: Headless 아키텍처 도입
- **개념**: 과거에는 유닛의 도면 형태(SVG 경로)를 `floorPlanLayouts.ts` 라는 별도의 코드 파일에 하드코딩하여 관리했습니다. **Headless 아키텍처**는 이 구조를 완전히 뒤집어, 모든 SVG `pathData`를 `TenantUnit` 객체의 속성으로서 데이터베이스에 직접 저장하고 관리하는 방식입니다.
- **핵심 변화**: 이제 UI 컴포넌트(`FloorPlan.tsx`, `FloorPlanDrafter.tsx` 등)는 도면의 '모양'을 알지 못합니다. 그저 데이터베이스로부터 받은 `pathData` 문자열을 화면에 '그려주기만' 하는 순수한 데이터 소비/생산자가 되었습니다.
- **기대 효과**:
    - **유지보수 비용 제로**: 과거에는 도면 수정 시 개발자의 코드 수정과 재배포가 필수였으나, 이제는 관리자가 '도면 편집기'에서 직접 수정하고 저장하면 즉시 모든 서비스에 반영됩니다. 개발자의 개입이 전혀 필요 없습니다.
    - **유연한 확장성**: 신규 건물을 추가하거나 기존 층의 레이아웃을 완전히 변경할 때, 코드 수정 없이 데이터베이스에 새로운 유닛 정보와 `pathData`를 추가하는 것만으로 시스템 확장이 가능합니다.
    - **안정성 및 일관성**: 모든 도면 데이터의 출처가 데이터베이스로 단일화되어, 데이터 불일치 문제가 원천적으로 차단됩니다.

## 10. 신규 기능: 공실 해소 시뮬레이터 (2024.05.20 추가)

### 10.1. 기능 개요

'실시간 임대 현황 및 수익 분석' 대시보드에 추가된 기능으로, 사용자가 **가상의 임대 시나리오**를 설정하여 그에 따른 **수익 및 임대율 변화를 즉각적으로 예측**할 수 있도록 지원합니다. 이를 통해 정적인 데이터 확인을 넘어, 능동적인 의사결정을 돕는 'What-if 분석' 환경을 제공합니다.

### 10.2. 기술적 구현

#### 1. 분석 로직 확장 (`src/hooks/useLeaseAnalytics.ts`)

- **`calculateLeaseAnalytics` 함수 분리**: 기존 `useMemo` 훅 내부에 있던 핵심 분석 로직을 별도의 순수 함수 `calculateLeaseAnalytics`로 분리하여, 현재 데이터 분석과 시뮬레이션 로직에서 모두 재사용할 수 있도록 구조를 개선했습니다.
- **`runLeaseSimulation` 함수 신설**:
    - **입력**: `areaToLease` (사용자가 추가로 임대하고자 하는 면적), `avgRentPerSqm` (평당 평균 임대료)
    - **프로세스**:
        1.  `JSON.parse(JSON.stringify(units))`를 통해 원본 데이터의 불변성을 보장하며 가상의 `simulatedUnits` 배열을 생성합니다.
        2.  공실 상태인 유닛 중 면적이 작은 순서대로 정렬하여, `areaToLease`에 도달할 때까지 순차적으로 상태를 `'입주'`로 변경하고 가상의 임대료를 책정합니다.
        3.  가상 데이터(`simulatedUnits`)를 `calculateLeaseAnalytics` 함수에 전달하여, 시뮬레이션 후의 임대 현황 데이터를 계산합니다.
    - **출력**: 시뮬레이션이 적용된 후의 `LeaseAnalytics` 객체

#### 2. 시뮬레이터 UI (`src/components/LeaseSimulator.tsx`)

- **개요**: 사용자가 시뮬레이션 조건을 입력하고 결과를 확인할 수 있는 UI 컴포넌트입니다. `Card` 내부에 `Slider`와 `Button`을 배치하여 상호작용성을 높였습니다.
- **핵심 UI 요소**:
    - **`Slider` 컴포넌트**: 사용자가 추가 임대 면적을 직관적으로 조절할 수 있도록 합니다. `@radix-ui/react-slider`를 기반으로 하는 커스텀 UI 컴포넌트(`src/components/ui/slider.tsx`)를 생성하여 사용했습니다.
    - **결과 표시**: 시뮬레이션 실행 후, 원본 데이터와 가상 데이터를 비교하여 '예상 임대율'과 '월 예상 수익'의 변화량을 `+N%p`, `+N억` 형태로 명확하게 보여줍니다.

#### 3. 페이지 통합 (`src/pages/LeaseAnalysisPage.tsx`)

- **레이아웃 변경**: 기존의 단일 레이아웃을 `lg:grid-cols-3`의 2단 그리드 구조로 변경했습니다.
- **컴포넌트 배치**: 좌측(2/3)에는 기존의 핵심 지표와 차트를, 우측(1/3)에는 새로 생성한 `LeaseSimulator` 컴포넌트를 배치하여 사용자가 현황과 예측을 동시에 파악할 수 있도록 UX를 개선했습니다.
