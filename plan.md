### 데이터 무결성 복원 계획

**목표:** 초기 목업 데이터가 사용자 데이터와 섞이는 문제를 해결하여, 임대 및 세대 관리 화면에 정확한 계약 정보만 표시되도록 합니다.

**핵심 전략:** 목업 데이터와 실제 사용자 데이터를 분리하고, 더 이상 신규 사용자에게 목업 데이터가 제공되지 않도록 시스템을 수정합니다.

---

#### 1단계: 초기 데이터 분리 (Seed Data Isolation)

- **작업:** `initial-tenants.ts`와 `initial-contracts.ts` 파일의 내용을 빈 배열로 변경합니다.
- **이유:** 신규 사용자가 가입할 때, 시스템이 더 이상 불필요한 목업 데이터를 Firestore에 저장하지 않도록 합니다. 이를 통해 새로 생성되는 모든 사용자 계정은 깨끗한 상태에서 시작하게 됩니다.
- **파일:**
  - `src/data/initial-tenants.ts`
  - `src/data/initial-contracts.ts`

---

#### 2단계: 데이터 로딩 로직 수정 (Data Loading Logic Modification)

- **작업:** `ProjectDataProvider.tsx`에서 Firestore 데이터 로딩 로직을 수정합니다.
- **이유:** 기존에는 Firestore에 데이터가 없을 경우, 로컬 목업 데이터를 기본값으로 사용했습니다. 이 부분을 수정하여, 데이터가 없는 경우에는 빈 배열을 사용하도록 변경합니다. 이는 데이터가 Firestore에만 의존하도록 하여, 로컬 목업 데이터와의 의존성을 완전히 제거합니다.
- **파일:** `src/providers/ProjectDataProvider.tsx`

---

#### 3단계: 기존 사용자 데이터 정제 (Existing User Data Cleanup)

- **작업:** 기존 사용자들의 Firestore 문서에서 중복되거나 오래된 목업 데이터를 식별하고 제거하는 마이그레이션 스크립트를 작성하고 실행합니다.
- **이유:** 이미 목업 데이터와 섞인 기존 사용자들의 데이터를 정상화하여, 모든 사용자가 정확한 데이터를 볼 수 있도록 보장합니다. 이 작업은 일회성으로 진행되며, 시스템이 안정화된 이후에는 필요하지 않습니다.
- **파일:** `scripts/migration-v2.ts` (신규 생성)

---

#### 4단계: 테스트 및 검증 (Testing and Validation)

- **작업:** 모든 변경 사항이 적용된 후, 임대 및 세대 관리 화면을 포함한 모든 관련 기능이 정상적으로 작동하는지 확인합니다.
- **이유:** 수정된 로직이 의도한 대로 작동하는지, 그리고 부작용은 없는지 최종적으로 검증합니다. 특히, 신규 사용자 가입, 기존 사용자 데이터 표시, 데이터 추가/수정/삭제 기능을 집중적으로 테스트합니다.
